# 公众号爆款公式速查表

> 本文件是分析完文章后辅助输出"可复用套路"的参考库。
> 分析时可从本表中比对文章用了哪套公式，也可在套路总结环节直接引用对应模板。

## 一、标题公式库（按场景分类）

### 场景：信息/资讯类

| 公式名 | 结构 | 示例 |
|--------|------|------|
| 权威背书型 | [机构/人名] + [动作] + [热点词] | 「工信部预警 OpenClaw！」 |
| 同天多件事 | 「同一天，发生了三件事」 | 适合新闻密集日的盘点 |
| 数字锚定型 | [具体数字] + [超预期结论] | 「3000行代码，反杀 43 万行」 |

### 场景：教程/工具类

| 公式名 | 结构 | 示例 |
|--------|------|------|
| 省钱直给 | 「别买 X 了！」+ 替代方案 | 「别买 API 了！」 |
| 保姆级承诺 | 标题末尾加「保姆级教程」 | 降低技术恐惧者的点击阻力 |
| 收藏价值型 | 末尾加「建议收藏」或「附XX方案」 | 给读者一个存档的理由 |

### 场景：观点/情感类

| 公式名 | 结构 | 示例 |
|--------|------|------|
| 反常识颠覆 | 「其实，[反直觉结论]」 | 「「小」才是正确答案」 |
| 身份代入 | 「给[目标群体]的一封信」 | 精准锁定目标读者 |
| 荒诞拼接 | [A] + [B] 两个本不相干的词 | 「龙虾公务员已入编」 |

## 二、开头公式库

### 公式 1：矛盾对冲型（最通用）

```
[正向事实，数字或结论]。
但，[制造矛盾的事实]。
```

适用：技术、职场、行业趋势类文章
效果：在30字内制造问题悬置，推动读者继续读

---

### 公式 2：场景递进型（适合热点破圈题材）

```
[这件事]，出圈了。
不是在[预期场所]。
是在[出人意料的地方]。
```

适用：描述某个事物扩散范围超出预期
效果：用反转给读者"原来已经这么大了"的震惊感

---

### 公式 3：问题前置型（适合痛点类内容）

```
你有没有遇到过这种情况：
[描述一个具体的痛苦场景]。
我们今天来解决这个问题。
```

适用：解决方案类、教程类
效果：先确认读者的痛，再给出口，建立"作者懂我"的信任

---

### 公式 4：承诺兑现型（适合有粉丝基础的账号）

```
答应你们的[内容类型]，来了。
```

适用：账号已有粉丝期待，在上篇文章留过悬念
效果：即时满足感，建立账号"说话算话"的人设

---

### 公式 5：数字冲击型（适合数据密集的行业报道）

```
[令人震惊的数字]。
[第二个对比数字]。
[解释这两个数字意味着什么]。
```

适用：报道效果、增长、对比类内容
效果：让读者在开头就感受到量级差异，为后续论证铺垫

## 三、情绪曲线设计公式

### 模型 A：焦虑-解法型（最常见）

```
第一段：制造问题（你可能有这个风险/这件事你不知道）
中段：放大问题（数据+场景证明问题很严重）
后段：给解法（具体可操作的步骤）
结尾：行动号召（现在就可以做）
```

适用：安全、健康、职业发展、工具推荐类

---

### 模型 B：反转-惊喜型

```
第一段：呈现一个"你以为是这样"的场景
中段：逐步揭示"其实不是这样"
后段：给出新的认知框架
结尾：用简洁语言封装这个新认知
```

适用：分析类、观点类、反常识内容

---

### 模型 C：叙事-共鸣型（适合情感类/故事类）

```
第一段：具体的人物/场景（不是"很多人"，是"某个具体的人"）
中段：事件推进，情绪随之起伏
转折点：一个出乎意料的细节或结果
结尾：从个体故事抽象出普遍性的感受或结论
```

适用：人物故事、职场叙事、生活观察类

---

### 模型 D：信息密集型（适合资讯/工具账号）

```
第一段：结论前置（最重要的信息先说）
中段：分点展开（每段一个子信息，短句密集）
各段节奏：事实 → 影响 → 下一个事实
结尾：一句话总结+行动引导
```

适用：AI资讯、科技新闻、行业动态类

## 四、金句生成公式

### 公式 1：极端对比型

```
[A] 可能就花掉了[B]的[时间/金钱]。
```

示例：「做一次 Code Review 可能就花掉了别人一个月的钱。」
适用场景：价格/效率对比，制造价值感知冲击

---

### 公式 2：逻辑闭环型（三段式）

```
[问题的来源]。
[问题的解法]。
闭环了。/ 这就是答案。
```

示例：「AI 挖的坑，AI 自己填。AI 写的代码，AI 组团来审。闭环了。」
适用场景：有因果逻辑的总结，适合文末

---

### 公式 3：比喻浓缩型

```
[复杂的事] = [用一个生活化的行为描述]
```

示例：「养龙虾可以，先把笼子装好。」
适用场景：技术/规则/方法论内容的最终浓缩，适合结尾金句

---

### 公式 4：职场语境翻译型

```
从「[旧状态的职场比喻]」变成了「[新状态的职场比喻]」。
```

示例：「从「听指令的实习生」变成了「会自己主动干活的员工」。」
适用场景：描述技术或能力的升级，适合受职场人群体欢迎的内容

---

### 公式 5：窗口期金句型

```
趁[某个条件还成立]，[行动]。
```

示例：「趁现在门还开着，可以冲了。」
适用场景：稀缺性/时间敏感性内容，制造行动冲动

## 五、互动钩子设计公式

### 评论钩子：双向包容型

```
[成功的]或者[没成功的]，评论区[说一声/聊聊]。
```

核心逻辑：覆盖所有状态的读者，让每个人都有资格留言。
适用：教程类、操作类内容

---

### 评论钩子：经验征集型

```
你们在用[什么方法/工具]？评论区聊聊。
```

核心逻辑：让读者感到自己的经验有价值，同时帮作者收集下篇素材。
适用：有一定粉丝互动基础的账号

---

### 评论钩子：争议预埋型（高风险高收益）

```
[做出一个有争议的断言]。（文章结尾不加解释，让读者自己下场）
```

示例：「OpenClaw「大而全」，DinTal Claw「小而精」，政务场景谁更适合？」（不给答案）
核心逻辑：两个阵营的读者都有表达欲，评论区自然对立，平台算法会推流。
注意：争议要是真争议，不能是明显有对错的问题。

---

### 转发钩子：关系传递型

```
觉得有用，转发给你身边[目标群体]的朋友。
```

核心逻辑：给转发行为定义一个"帮朋友"的意义，不是帮账号，是帮朋友。
适用：工具、教程、安全提示类内容

---

### 转发钩子：身份宣告型（需要一个强 Slogan）

```
[账号 Slogan 出现在每篇结尾]
```

核心逻辑：读者转发这篇文章，朋友圈就会显示这个 Slogan，Slogan 本身就是读者愿意公开认同的身份标签。
设计要点：Slogan 要让读者觉得"我认同这句话描述的那种人"，而不是在帮你打广告。
好的示例：「让 AI 成为你的外挂」（认同这句话=我是拥抱AI的人）
弱的示例：「关注我获取最新AI资讯」（这是账号诉求，不是读者身份）

## 六、说服策略组合推荐

根据文章类型，推荐不同的说服策略组合：

**资讯/新闻类**：数据权威 + 场景还原 + 身份背书
→ 效果：可信度高，读者感觉在看一手信息

**教程/工具类**：利益直给 + 对比反差 + 操作化行动引导
→ 效果：降低门槛，读者感觉"我可以做到"

**观点/分析类**：反常识颠覆 + 案例佐证 + 开放式结尾（不给答案）
→ 效果：激发讨论欲，适合有争议性的主题

**安全/风险类**：场景恐惧 + 数据支撑 + 解法落地
→ 效果：让读者从"觉得有点道理"变成"我要现在去做"

## 七、公众号账号人设设计参考

爆款文章不只是单篇内容的成功，往往是账号人设在每篇文章里持续强化的结果。以下是常见的有效人设构成：

**信息差型人设**（适合资讯、行业类账号）
- 核心定位："我比你早知道，而且我会翻译给你听"
- 关键信号：标注学历/职业背景、使用"亲测"、"内部消息"等词
- 风险：一旦信息出错，信任崩塌迅速

**实操者人设**（适合工具、教程类账号）
- 核心定位："我是真的在用，不是理论"
- 关键信号：有具体的操作截图/命令/数字、踩过真实的坑
- 风险：需要持续输出高质量实操内容，容易断更

**向导型人设**（适合综合类AI/科技账号）
- 核心定位："你不需要懂技术，我帮你理解就够了"
- 关键信号：所有专业词汇都有口语化翻译、结尾有行动指引
- 风险：深度技术读者容易流失，定位在中间层
