# 输出质量检查清单

在交付前逐项检查；任何一项明显不满足，都先回退修正，再输出。

## 需求忠实度

- 是否区分了用户原话、已确认事实和 `[默认推断]`
- 是否把高影响不确定项收敛到“待确认项”
- 是否避免把会议纪要逐句复述成流水账

## 产品结构

- 是否明确了目标用户、核心目标和关键对象
- 是否给出了角色、模块、页面和主流程
- 是否补上了关键实体状态和权限边界

## 页面完整度

- 是否输出了真正的页面，而不是功能点清单
- 每个核心页面是否至少有 3 个组件
- 每个核心页面是否至少有 2 条交互
- 每个核心页面是否覆盖 4 类状态
- 是否为每个核心页面给出成体系的 mock data

## 渲染交接质量

- 是否明确了首批渲染页面和优先级
- 是否说明了导航模型、响应策略和视觉预设
- 是否限制在单文件 HTML + 原型层交互范围内
- 是否把需要遵守的技术边界写清楚

## 视觉质量

- 颜色是否符合对应行业气质，而不是随机选色
- 是否避免 Inter、紫色白底、廉价渐变和平均分配多色
- 是否让主色、强调色和状态色职责清晰
- 是否让页面层级、留白、阴影和字体保持一致

## 可交付性

- HTML 是否可直接浏览器打开
- Hash 路由是否可用
- 抽屉、弹窗、Toast 是否至少有一条可演示路径
- 首轮范围是否收敛，没有无关页面膨胀
