# 示例：会议纪要转原型

## 输入

今天先别做太复杂，先给客户看一个仓库调度系统的大概页面。
客户最关心整体运行状态，想看到当前任务数、异常数、设备在线率。
现场操作的人还要能看到任务列表，点进去看详情，最好能直接处理异常。
设备这块不用一开始做得太深，但要能看在线、离线、告警。
页面风格偏专业一点，不要太花。
先 PC 端后台，不做移动端。

## [需求解析摘要]

- **产品方向**：B 端仓储调度后台
- **目标用户**：调度主管、现场操作员
- **业务目标**：为客户演示整体运行态势和异常处理能力
- **关键对象**：调度任务、异常事件、设备
- **已确认事实**：
  - 首轮做 PC 端，不做移动端
  - 客户关心整体状态（任务数、异常数、设备在线率）
  - 现场操作员需要任务列表和异常处理能力
  - 设备首轮只看状态，不做深层管理
  - 风格偏专业克制
- **[默认推断]**：
  - `[默认推断]` 调度主管主要看总览，操作员主要用列表和详情
  - `[默认推断]` 异常处理通过抽屉内提交表单完成
  - `[默认推断]` 设备状态作为总览页的一个区域，不单独成页
- **待确认项**：
  1. 是否需要角色切换或权限区分？
  2. 异常处理后是否需要审批流程？

## [页面设计方案]（节选：总览页）

### 运行总览
- 页面目标：30秒内建立整体运行态势认知
- 适用角色：调度主管
- 页面布局：顶部筛选 → 三列指标卡 → 任务趋势图 → 双列（异常列表 + 设备状态）
- 核心区域：指标卡区、任务趋势区、异常列表区、设备状态区
- 核心字段 / 数据：在途任务数（128）、待处理异常（9）、设备在线率（96.2%）
- 主操作 / 次操作：点击卡片进入列表（主）、点击异常行打开抽屉（次）
- 交互行为：点击指标卡跳转列表页、点击异常行打开详情抽屉、筛选联动刷新指标
- 状态反馈：loading 骨架屏 / empty 暂无数据 / error 加载失败+重试 / no_permission 无权限
- 页面跳转关系：指标卡 → 任务列表页、异常行 → 异常详情抽屉
