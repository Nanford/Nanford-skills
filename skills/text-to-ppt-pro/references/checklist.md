# 交付前巡检清单

生成完成后，逐维度执行检查。发现问题时，按**修版优先级**修复，不要仅标记问题而不修复。

---

## 一、结构检查

- [ ] 整套 PPT 是否有封面、导读、主体、总结的完整链路
- [ ] 章节切换是否自然（过渡页是否到位）
- [ ] 是否存在内容重复页（相邻两页是否在讲同一件事）
- [ ] 每页是否都有清晰任务（能用一句话说清这页的目的）

---

## 二、文案检查

- [ ] 标题是否像结论，而不是像目录条目（❌"项目背景" → ✅"传统流程导致30%效率浪费"）
- [ ] 页面是否存在大段正文堆砌（单页超 80 字考虑拆页）
- [ ] 是否把适合备注的内容塞到了正文
- [ ] 关键词 / 关键数字是否足够醒目
- [ ] 专业术语是否需要补充解释

---

## 三、版式检查

- [ ] 是否存在文字溢出边框或超出安全区
- [ ] 是否存在遮挡、压线、挤压
- [ ] 卡片间距是否均衡（建议 16–28 px）
- [ ] 左右结构是否失衡（视觉重量差异过大）
- [ ] 页面视觉重心是否稳定
- [ ] 换行是否自然（不在关键词中间断行）
- [ ] 多行标题是否难看

---

## 四、视觉检查

- [ ] 整套颜色是否统一（只用选定主题的色值）
- [ ] 图标风格是否一致（全线性 or 全线面结合，不混搭）
- [ ] 是否有某页过度装饰（堆满图形、渐变、阴影）
- [ ] 是否有某页显得过空（主体面积不足 40%）或过满
- [ ] 相邻页面是否过于雷同（连续 5 页一个样式）

---

## 五、表达检查

- [ ] 观众是否能在 3 秒内抓到重点
- [ ] 页面是否支持讲师自然展开（备注区有提示）
- [ ] 是否存在"作者自己知道、观众看不懂"的表达
- [ ] 是否一页试图讲两个主题（应拆页）

---

## 六、交付检查

- [ ] 页码是否连续、格式统一
- [ ] 备注区是否写入讲师逐字稿或提示
- [ ] 文件命名是否清晰（主题_版本_日期）
- [ ] 是否输出了修正版或附总览表供复核

---

## 七、修版优先级

**当一页不好看时，按以下顺序修（不要跳过前面的步骤直接缩字）：**

1. **删减不必要的屏幕文字** — 能进备注区的不放正文
2. **调整内容分组** — 重新梳理信息层级
3. **改用更适合的页型** — 可能这一页根本不应该用这种版式
4. **重设栏宽与留白** — 调整网格比例，增加呼吸感
5. **把解释性内容挪到备注区** — 讲师讲，屏幕不讲
6. **最后才考虑缩小字号** — 且不低于正文 10.5 pt、标题 16 pt
