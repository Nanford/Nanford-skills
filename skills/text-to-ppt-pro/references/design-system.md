## 默认尺寸

- 画布比例：16:9
- 推荐尺寸：13.333 × 7.5 英寸 或 1920 × 1080 逻辑画布
- 页面安全边距：左右 0.55–0.75 英寸，上下 0.4–0.65 英寸

## 字体规范

### 字体栈

| 用途 | 推荐字体 |
|------|---------|
| 中文标题 | 微软雅黑 / 思源黑体 / HarmonyOS Sans SC / 阿里巴巴普惠体 |
| 中文正文 | 微软雅黑 / 思源黑体 |
| 英文/数字 | Inter / Aptos / Segoe UI |

### 字号层级

| 位置 | 字号范围 |
|------|---------|
| 封面主标题 | 26–34 pt |
| 页内主标题 | 20–28 pt |
| 副标题 | 11–16 pt |
| 正文 | 10.5–16 pt |
| 注释/页脚 | 8–10 pt |

### 字重策略

| 级别 | 字重 |
|------|------|
| 一级标题 | Bold / Semibold |
| 二级标题 | Semibold |
| 正文 | Regular |
| 强调数据 | Bold |

## 页面通用结构规则

### 标题区
- 顶部保留统一标题区
- 标题与正文之间留 18–28 px 视觉呼吸感
- 标题过长时，优先拆副标题，不优先缩字

### 内容区
- 优先采用网格系统
- 左右分栏比例建议：5:5 / 6:4 / 7:5
- 卡片间距建议：16–28 px
- 不同信息块必须有明显边界或留白区隔

### 页脚区
- 可放页码、日期、项目名称、品牌名
- 页脚应弱化，不抢正文重心

---

## 图标与图形建议

- 图标统一线性或线面结合风格，不混搭
- 优先使用简洁图标，不使用花哨插画
- 辅助图形只用于增强节奏，不可喧宾夺主
- 科技感页面可使用弱化点线网络、几何底纹、微渐变发光

---

## 图表规则

- 不要为了"像 PPT"强行塞复杂图表
- 图表只服务于结论，不服务于装饰
- 一页只表达一个主要结论
- 图表旁必须配结论短句或关键数字

---

## 留白原则

- 不要把"放得下"误判为"排得好"
- 页面越复杂，越需要留白分区
- 过渡页、封面页、大结论页，要敢于留白

---

## 版式节奏原则

整套 PPT 内必须有节奏变化：

| 页型 | 核心任务 |
|------|---------|
| 开场页 | 重定调 |
| 导读页 | 重预期 |
| 过渡页 | 重节奏 |
| 框架页 | 重认知 |
| 方法页 | 重结构 |
| 案例页 | 重代入 |
| 总结页 | 重收束 |

**不要连续 5 页都长得像同一页。**
