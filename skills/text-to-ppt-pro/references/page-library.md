# 页面类型库 — 13 种标准页型

## 页型选择原则

规划每页前先问：
- 这页是为了让人"听懂"，还是"看懂"？
- 这页是建立认知，还是推动决策？
- 这页是信息呈现，还是章节节奏？
- 这页观众看完后，应该记住什么？

**先判断功能，再选页型。**

---

## 1. 封面页 `cover`

**适用：** 开场、课程封面、汇报题面

**特点：** 大标题、强定调、极简信息

**组成：**
- 主标题（结论式，不是课题名）
- 副标题（一句话说明用途或核心价值）
- 讲者 / 日期 / 单位
- 弱化背景图形（几何底纹、微渐变）

**禁止：** 堆砌多段文字、放目录、放大量 Logo

---

## 2. 导读页 `intro`

**适用：** 说明为什么讲、解决什么问题、拉齐预期

**结构：** 左右对照 或 上下递进

| 左侧 | 右侧 |
|------|------|
| 常见误区 / 现状问题 | 本次目标 / 收益 |
| 观众带来的问题 | 本课程的解决方向 |
| 旧的认知框架 | 新的认知方式 |

---

## 3. 目录页 `toc`

**适用：** 章节预览，3–6 个章节

**结构：**
- 四段式目录（编号 + 标题 + 一句话说明）
- 当前章节可用强调色高亮
- 每项不超过一行，保持扫视友好

---

## 4. 章节过渡页 `transition`

**适用：** 切换模块、标记章节边界

**特点：**
- 极简、居中、大字
- 章节编号（如 `01 / 认识 AI`）
- 一句话副标题说明本章核心
- **不要堆内容**，留白为主

---

## 5. 四宫格认知页 `four-grid`

**适用：** 概念分层、能力模型、角色定义、象限分析

**结构：** 2×2 卡片 或 4 列卡片

每格包含：
- 标题（关键词）
- 一句话定义
- 必要时一条应用说明

**禁止：** 每格塞多段文字，所有格等权重（应有主次）

---

## 6. 对比页 `comparison`

**适用：** 前后对比、方案对比、误区与正确方式、旧路vs新路

**结构：** 左右对照，视觉对立

| 左侧 | 右侧 |
|------|------|
| 问题 / 旧方式 / 误区 | 目标 / 新方式 / 正解 |
| 用警示色（danger/warning）弱化 | 用强调色高亮 |

**要求：** 两侧维度项数一致，对齐对照

---

## 7. 流程页 `process`

**适用：** 步骤、机制、链路、业务流、审批流

**结构选项：**
- 横向流程：3–6 步从左到右
- 纵向流程：超过 6 步改竖排
- 泳道图：多角色协作流程
- 步骤编号卡片：强调顺序

**要求：**
- 每一步只保留关键词（2–5 字）
- 复杂解释进备注，不塞步骤框
- 步骤间用箭头或连接线

---

## 8. 方法步骤页 `method`

**适用：** SOP、落地方法、实施路径、操作指南

**结构：**
- 编号步骤（01 / 02 / 03 格式）
- 左侧步骤名，右侧说明（左窄右宽，5:5 或 4:6）
- 或 3步 / 5步 线性链路（上名称+下说明）

**与流程页的区别：** 方法页强调"如何做"，流程页强调"怎么走"

---

## 9. 框架模型页 `framework`

**适用：** 能力地图、认知框架、分层体系、系统架构概览

**结构选项：**
- 同心圆：由内而外的层级关系
- 金字塔：层级递进、基础→高级
- 分层卡片：各层横向铺开
- 多列框架图：并列维度对比

**要求：** 图形必须可读，不为了"好看"牺牲可读性。配关键词标注。

---

## 10. 案例页 `case-study`

**适用：** 案例讲解、场景说明、实践举例

**推荐结构（选其一）：**

**结构 A — 问题-动作-结果：**
| 问题 | 动作 | 结果 |
|------|------|------|
| 面临什么挑战 | 做了什么 | 取得什么成果 |

**结构 B — 背景-做法-价值：**
| 背景 | 做法 | 价值 |
|------|------|------|
| 客户是谁/在什么情况 | 我们怎么做的 | 带来了什么 |

**要求：** 数据量化结果，不用模糊描述

---

## 11. 数据结论页 `data`

**适用：** 经营数据、实验结果、评估结果、KPI展示

**布局原则：**
- **结论写在图的上方**（不是图下方）
- 关键数字做放大呈现（大号 Bold，强调色）
- 图表只支撑一个主要观点
- 一页只有一张主图

**禁止：** 图表旁没有结论句、一页放多张图、数字没有单位

---

## 12. 总结页 `summary`

**适用：** 模块收束、全篇收束、行动建议

**结构：**
- 3 个结论（最多）
- 3 个行动建议
- 或"记住这三件事"

**禁止：** 把总结页做成重复内容的堆砌，禁止超过 5 个要点

---

## 13. 致谢页 `closing`

**适用：** 最后一页，Q&A 引导

**特点：** 简洁、克制、不花哨

**组成：**
- 感谢语（1–2 行）
- 联系方式（可选）
- 品牌标识（可选）
- Q&A 提示（可选）

---

## 典型场景的页型组合

### 培训课件
```
封面 → 导读 → 目录 → 过渡 → 四宫格/框架 → 方法步骤 × N → 案例 → 总结 → 致谢
```

### 项目汇报
```
封面 → 执行摘要(论点) → 目录 → 过渡 → 背景/问题 → 阶段成果(数据) → 风险对比 → 计划流程 → 总结 → 致谢
```

### 售前方案
```
封面 → 导读(行业痛点) → 对比(现状vs目标) → 框架(方案架构) → 功能/案例 × N → 实施路径 → 价值数据 → 总结 → 致谢
```

### 知识型演示
```
封面 → 导读 → 目录 → 过渡 × N → 框架 → 方法 × N → 案例 → 总结 → 致谢
（不强行满屏图形，控制每页文字量，卡片分组承载要点，大段说明进备注区）
```
