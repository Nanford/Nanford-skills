# text-to-ppt-pro

你的任务不是把文字塞进幻灯片，而是把用户内容转化为**逻辑清晰、风格统一、层级明确、可直接交付**的专业 PPT。

## 输入理解

收到内容后，先识别：
- 主题 / 受众 / 用途（讲解 / 汇报 / 说服 / 培训 / 宣贯）
- 是否已有逐页结构
- 是否有讲师逐字稿 / 备注
- 风格偏好（科技感 / 商务感 / 极简 / 麦肯锡等）
- 优先"视觉高级感"还是"表达清晰度"

信息不完整时，根据上下文合理推断，不轻易中断流程。

---

## 执行流程

### 1. 定全局系统

先确定：
- 页面比例：默认 **16:9**，推荐画布 13.333 × 7.5 英寸
- 主题预设：`dark_tech` / `business_blue` / `mckinsey_clean` / `warm_exec`（详见 `references/design-system.md`）
- 字体层级、强调色、卡片规则、留白尺度
- 备注区是否启用

**主题快速选择：**

| 场景 | 推荐主题 |
|------|---------|
| AI培训、技术方案、产品演示 | `dark_tech` 深色科技感 |
| 项目汇报、售前提案、甲方汇报 | `business_blue` 商务蓝白 |
| 管理复盘、策略汇报、领导简报 | `mckinsey_clean` 麦肯锡简洁风 |
| 董事会汇报、经营分析、年度总结 | `warm_exec` 暖灰高管汇报 |

### 2. 逐页判断页面角色

**每页必须先定类型，再决定排版。严禁不分类直接套模板。**

13 种页面类型（详见 `references/page-library.md`）：

| 类型 | 关键词 |
|------|--------|
| 封面页 | 大标题、定调、极简 |
| 导读页 | 为什么讲、解决什么问题 |
| 目录页 | 编号+标题+章节预览 |
| 章节过渡页 | 极简、大字、节奏切换 |
| 四宫格认知页 | 概念分层、2×2卡片 |
| 对比页 | 左右对照、误区vs正解 |
| 流程页 | 横向/纵向/泳道/编号 |
| 方法步骤页 | SOP、编号步骤、实施路径 |
| 框架模型页 | 同心圆、金字塔、分层体系 |
| 案例页 | 问题-动作-结果 |
| 数据结论页 | 结论在上、图表支撑 |
| 总结页 | 3个结论/建议 |
| 致谢页 | 简洁克制 |

页型选择前问自己：这页为了"听懂"还是"看懂"？建立认知还是推动决策？

### 3. 内容三层分层

| 层级 | 内容 | 处理规则 |
|------|------|---------|
| 第一层 | 观众 3 秒内必须看到的结论 | **必须上屏** |
| 第二层 | 支撑结论的关键点 | 按密度选择上屏或收纳 |
| 第三层 | 适合讲解展开的补充内容 | **优先进备注区**，不强塞正文 |

### 4. 版式匹配

| 内容密度 | 版式选择 |
|---------|---------|
| 信息少 | 极简大字 + 强留白 |
| 信息中等 | 左右对比 / 上下递进 / 卡片布局 |
| 信息复杂 | 流程链路 / 分区收纳 / 矩阵 / 编号步骤 |

**禁止通过缩小字体解决内容过多问题。**

### 5. 统一视觉语言

整套必须统一：背景、字体层级、强调色、卡片规则（圆角/阴影/透明度）、图标风格、页脚规则。版式节奏：开场定调 → 导读预期 → 过渡节奏 → 框架认知 → 方法结构 → 案例代入 → 总结收束。**不要连续 5 页都长得像同一页。**

### 6. 备注区策略

用户提供讲师逐字稿、演讲提词、扩展说明 → 写入备注区。页面只保留观众必须看到的信息。

### 7. 巡检与修复

生成后逐页检查：文字溢出 / 元素遮挡 / 换行难看 / 卡片间距失衡 / 视觉重心偏移 / 过满过空 / 连续重复版式。

**修复优先顺序（按序执行）：**
1. 精简正文
2. 调整栏数或结构
3. 提高留白
4. 内容下沉至备注区
5. 重做页面角色匹配
6. 最后才考虑缩小字号

## 高质量标准

- 每页都有明确任务
- 观众 3 秒内能抓到重点
- 相邻页面有节奏变化
- 层级清晰，不靠堆字解释
- 整套风格稳定，不像模板拼盘
- 无错乱、溢出、遮挡、失衡
- 讲师能根据备注顺畅讲解

---

## 修版模式

用户反馈"有些页面乱了"/"排版不好看"/"内容挤"/"层级不清"时，立即进入修版模式：
逐页巡检 → 标记问题页 → 分类问题类型 → 重排结构 → 输出修正版。

---

## 程序化生成

用户需要直接生成可编辑 `.pptx` 文件时，参阅 `references/script-reference.md`，支持：
- **Python + python-pptx**：内容结构清晰、批量插入、与文档处理联动
- **Node.js + PptxGenJS**：精确布局控制、多图表图形、稳定生成器

最佳实践：Skill 拆结构与页型 → 脚本快速生成初稿 → 一轮巡检和精修。
