# 五种正文结构详解

## 1. 观点驱动型

**适用场景**：热点评论、趋势判断、行业分析

**结构模板**：

```
[开头：反常识/认知冲击/趋势预警，100字内抓住读者]

[第一段：快速交代发生了什么，但重点是"为什么你应该继续看"]

[第二段：抽出本质变化——不是"发生了什么"，而是"这意味着什么"]

[第三段：分人群影响——不同角色分别受什么影响，机会点/风险点在哪里]

[第四段：情绪拔高——金句、刺痛句、情绪价值句集中爆发]

[结尾：升华观点 + 互动钩子]
```

**写作要点**：
- 核心观点必须是判断性命题，不是事实陈述
- 第三段是价值最高的部分，要给出明确判断，不要"也许""可能"
- 情绪节奏：好奇 → 理解 → 被说服 → 被触动

---

## 2. 故事驱动型

**适用场景**：人物报道、案例复盘、经历分享、产品故事

**结构模板**：

```
[场景切入：把读者拉进一个具体的场景或时刻]

[故事推进：按时间线或因果链展开，保持细节感]

[转折点：故事中的关键变化、意外发现或决策时刻]

[提炼洞察：从故事中抽出普适性的认知——"这个故事告诉我们什么"]

[升华主题：把个案上升到趋势/规律层面]

[结尾：回扣开头场景 + 互动钩子]
```

**写作要点**：
- 场景切入要有画面感，让读者"看到"而不是"听到"
- 故事不能流水账，每个段落都要推进情节或认知
- 转折点是全文的情绪峰值，要写得有力
- 洞察提炼不能牵强，必须从故事中自然生长出来

---

## 3. 对比驱动型

**适用场景**：产品评测、方案选择、新旧对比、路线之争

**结构模板**：

```
[抛出选择困境：读者正面临的选择难题，制造代入感]

[维度一对比：在第一个关键维度上，A 和 B 分别表现如何]

[维度二对比：在第二个关键维度上的差异]

[维度三对比：在第三个关键维度上的差异（可选更多维度）]

[揭示关键差异：这些对比背后，真正的分水岭是什么]

[给出判断：不同人群该怎么选，作者自己的选择和理由]

[结尾：互动钩子——"你选哪个？"]
```

**写作要点**：
- 不要做"功能清单式"对比，要有观点渗透
- 每个维度对比后加一句判断，不要让读者自己去总结
- "关键差异"那段是全文最有价值的部分，要写出别人没写到的洞察
- 结尾天然适合争议式互动钩子

---

## 4. 清单驱动型

**适用场景**：方法论总结、工具推荐、经验清单、避坑指南

**结构模板**：

```
[痛点切入：读者正在经历的困境，或一个常见误区]

[要点一：标题式小结 + 展开说明 + 实操建议或案例]

[要点二：同上结构]

[要点三：同上结构]

[……可根据需要增减，建议3-7个要点]

[总结提炼：这些要点背后的共同规律是什么]

[结尾：互动钩子——"你还有什么补充？"]
```

**写作要点**：
- 每个要点的小标题本身就要有信息量，不能是"第一点""第二点"
- 要点之间要有递进或逻辑关系，不能是随机堆砌
- 每个要点都要有"为什么"而不只是"是什么"
- 适合在要点中穿插金句和刺痛句

---

## 5. 争议驱动型

**适用场景**：有明确正反立场的话题、行业争论、价值观碰撞

**结构模板**：

```
[抛出争议：一个让人立刻想选边站的问题或现象]

[正方论据：支持 A 观点的理由，写得有说服力]

[反方论据：支持 B 观点的理由，同样写得有说服力]

[第三视角（可选）：跳出二元对立，提供更高维度的看法]

[作者判断：明确表态，给出自己的立场和理由]

[结尾：互动钩子——把争议抛给读者]
```

**写作要点**：
- 正反方论据都要写得有力，不能为了作者立场故意弱化对方
- 读者在"作者判断"之前应该感到"两边都有道理"
- 作者判断要果断，不要和稀泥
- 这种结构天然互动率高，结尾钩子要充分利用争议

---

## 结构选择速查

| 素材特征 | 推荐结构 |
|----------|----------|
| 有明确新事件/新发布 | 观点驱动型 |
| 有具体人物/案例/经历 | 故事驱动型 |
| 涉及两个或多个选项的比较 | 对比驱动型 |
| 可拆解为多个独立要点 | 清单驱动型 |
| 话题有明显正反立场 | 争议驱动型 |
| 不确定时 | 默认用观点驱动型 |
