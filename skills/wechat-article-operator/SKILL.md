# 公众号内容生产系统（V3）

## 定位

本 Skill 覆盖公众号文章从选题到成稿的完整链路：选题评估 → 读者锚定 → 结构选型 → 正文写作 → 传播元素植入 → 运营拆解。核心目标是让同样一个话题，写出更有传播力的版本。

---

## 一、人设配置（可选）

如果用户定义了账号人设，所有输出必须与人设保持一致。人设影响三个维度：

- **语气**：犀利型 / 理性温和型 / 轻松幽默型 / 导师型
- **立场倾向**：技术乐观派 / 审慎观察者 / 实用主义者 / 批判思考者
- **标志性表达**：账号特有的口头禅、句式习惯、段落节奏偏好

用户未指定人设时，默认使用"理性但有锋芒的行业观察者"风格：专业不端着，有判断但不偏激，有情绪但不油腻。

---

## 二、执行工作流

### Step 0：选题评估

在动笔前先做选题判断，给出明确的可写性结论。

**联网搜索（热点/新闻/产品更新类必须执行）**：使用 web_search 搜索主题相关的最新信息，确认事实准确性、是否有更新进展、是否有值得引用的差异化角度。工具测评类或纯个人观点类可跳过。

**选题五维评估**（每项给出高/中/低判断，1-2句理由）：

| 维度 | 判断标准 |
|------|----------|
| 时效性 | 是否在信息窗口期内？过了热度期的话题需要找新切入角度 |
| 话题饱和度 | 别人写过多少了？如果已经铺天盖地，必须找到差异化角度才值得写 |
| 读者价值 | 读者看完能得到什么？（省时间/开眼界/降焦虑/获判断/学方法） |
| 本质变化 | 这件事背后有没有值得提炼的深层趋势或结构性变化？ |
| 差异化空间 | 能不能写出和已有文章不同的观点、角度或信息增量？ |

评估结论分三档：
- **强选题**：直接进入写作流程
- **可写但需找角度**：先确定差异化切入点，再进入写作
- **弱选题**：向用户说明原因，建议换角度或换话题

---

### Step 1：读者画像锚定

明确"写给谁"，这决定了语气、痛点、信息深度和互动策略。

从以下维度快速锚定目标读者：

- **身份**：普通用户 / 创作者 / 开发者 / 产品经理 / 创业者 / 职场人
- **认知水平**：入门（需要科普）/ 进阶（需要洞察）/ 专业（需要判断）
- **核心诉求**：跟上热点 / 学到方法 / 获得判断 / 缓解焦虑 / 找到机会

用户未指定时，根据话题自动推断最合适的读者画像，并在输出中注明。

不同画像的写作调整：
- 入门读者：多类比、少术语、重"和我有什么关系"
- 进阶读者：可用行业术语、重本质分析和趋势判断
- 专业读者：可直接进入深度讨论、重差异化观点和前瞻性

---

### Step 2：确定文章主轴（核心观点）

确定一句话：读者看完之后最应该记住什么？

核心观点不是事实陈述（"某产品发布了新功能"），而是判断性命题（"这次更新标志着 AI 助手终于从聊天工具变成了真正意义上的执行层"）。

副观点不超过两个，用来支撑和丰富核心观点，不要分散主线。

---

### Step 3：选择正文结构

根据素材类型和话题特征，从以下结构中选择最适合的一种（不要每篇都用同一种）：

**观点驱动型**（适合热点评论、趋势判断）
> 抓人开头 → 交代事实 → 抽出本质 → 分人群影响 → 情绪拔高 → 互动结尾

**故事驱动型**（适合人物报道、案例复盘）
> 场景切入 → 故事推进 → 转折点 → 提炼洞察 → 升华主题 → 互动结尾

**对比驱动型**（适合产品评测、方案选择）
> 抛出选择困境 → A vs B 对比 → 揭示关键差异 → 给出判断 → 互动结尾

**清单驱动型**（适合方法论、工具推荐、经验总结）
> 痛点切入 → N 个要点逐条展开 → 总结提炼 → 互动结尾

**争议驱动型**（适合有明确正反立场的话题）
> 抛出争议 → 正方论据 → 反方论据 → 作者判断 → 互动结尾

选定结构后，在写作前简要说明选择理由。

---

### Step 4：设计开头抓手

开头不能平铺直叙。前 100 字内必须让读者产生"这篇值得继续看"的感觉。从以下切入方式中至少选择两种组合使用：

- **反常识**：先说一个大多数人以为是对的判断，然后翻转
- **用户痛点**：直接戳中目标读者此刻的焦虑或困惑
- **趋势预警**：暗示一件正在发生的变化，读者如果不了解会吃亏
- **情绪共鸣**：让读者在第一段就感到"这说的是我"
- **认知冲击**：给出一个让人意外的数据、对比或结论
- **场景代入**：描述一个读者熟悉的场景，然后引出反转

---

### Step 5：正文写作

按选定的结构推进正文，同时在写作过程中自然植入传播元素（在写的时候就埋进去，不要写完后单独补）：

**金句（3-5句）**：适合被读者截图、划线、转发的句子。要有观点力度，不能空洞，不能鸡汤。

**情绪价值句式（3-5句）**：让读者感到被理解、被鼓舞、被点醒的句子。

**刺痛句式（3-5句）**：点出用户的焦虑、误区、迟钝、行动拖延。要"刺痛但有认知"，不能只是批评。

---

### Step 6：输出运营拆解

正文完成后，输出结构化的运营拆解，供复盘和二次创作使用。

---

## 三、深度档位

根据文章定位选择档位，用户可指定，未指定时自动判断：

| 档位 | 字数 | 适用场景 |
|------|------|----------|
| 快评 | 500-800 字 | 热点快速响应、短观点输出 |
| 标准 | 800-1500 字 | 大多数公众号文章的最佳区间 |
| 深度 | 1500-3000 字 | 深度解读、长测评、趋势分析 |

不同档位的写作密度要求：
- 快评：一个核心观点，不展开副线，节奏快
- 标准：一主两副，有完整的情绪节奏设计
- 深度：可以有多层论证、多角度拆解、案例穿插

---

## 四、默认输出结构

### 1. 选题评估（快速模式可省略）

五维评估表 + 结论。

### 2. 标题（3个）

每次给出三个风格不同的标题版本：
- **爆款传播型**：有明显点击欲望，让人一眼想转发
- **观点犀利型**：体现作者判断力，让懂行的人觉得"这个人有想法"
- **专业分析型**：信息量大，适合偏理性读者和职场人群

标题不得使用虚假承诺或低俗夸大，但可以有情绪张力。

### 3. 公众号正文

按选定的结构和档位输出正文。

行文风格：多用短中句交替制造节奏感，专业但不端着，有情绪但不油腻，有判断不是资讯搬运。避免写成新闻稿、学术报告或流水账。

### 4. 运营拆解

按以下维度逐条输出（每条1-2句概括）：
- 核心观点
- 副观点
- 读者画像
- 文章结构类型及选择理由
- 说服策略（用了哪些论证方式）
- 情绪触发点（在哪几处设计了情绪峰值）
- 金句提炼
- 情感曲线分析（开头/中段/结尾的情绪走向）
- 视角转化分析（从哪些不同视角切入话题）
- 语言风格特征

### 5. 情绪价值句式（3-5句）

从正文中提炼，或额外补充。让读者感到"被这篇文章理解了"。

### 6. 刺痛句式（3-5句）

从正文中提炼，或额外补充。让读者感到"被说中了"或"不自觉想反驳"。

### 7. 互动钩子（3个）

三种风格各一个：
- **提问式**：让读者发表具体立场
- **争议式**：设置观点分歧，引发正反讨论
- **填空式**：留出空白让读者补充自己的经历

---

## 五、快速模式

当用户只给了一个主题或一句话，没有详细素材时：先主动 web_search 补充信息，跳过选题评估的详细输出，直接按默认输出结构输出。缺少的变量按最合理的方式自行补全，在运营拆解后注明补全了哪些假设。

---

## 六、改稿模式

当用户提供已有文章请求升级时，先做诊断再改稿。

**诊断维度**（逐条给出当前状态和改进方向）：
- 标题吸引力
- 开头钩子感
- 观点清晰度
- 信息 vs 观点比例
- 情绪节奏起伏
- 金句密度
- 结尾互动设计

**改稿输出**：优化后的标题（3个）→ 优化后的正文 → 本次优化说明（诊断结果 + 强化了哪些传播点）→ 情绪价值句式 → 刺痛句式 → 互动钩子。

---

## 七、发布前检验清单

- [ ] 选题有差异化角度，不是重复搬运
- [ ] 读者画像清晰，语气和痛点与目标读者匹配
- [ ] 开头 100 字内有明确的抓手，不是平铺直叙
- [ ] 正文中有明确的核心判断，不只是资讯罗列
- [ ] 正文写了"为什么重要"和"和读者有什么关系"
- [ ] 全文有情绪节奏起伏，不是平的流水账
- [ ] 埋入了 3 句以上适合划线的金句
- [ ] 有情绪价值句式（被理解/被鼓舞）
- [ ] 有刺痛句式（被说中/有张力）
- [ ] 结尾有互动钩子，不是只有总结
- [ ] 三个标题都准备好了
- [ ] 字数在选定档位范围内

## 八、工具链衔接

正文完成后，根据用户需要提示后续可用的工具：

- **排版**：使用 `wechat-article-formatter` skill 将 Markdown 正文转为微信公众号排版 HTML
- **发布**：使用 `baoyu-post-to-wechat` skill 直接发布到微信公众号
- **配图**：使用 `baoyu-cover-image` skill 生成封面图，或 `baoyu-article-illustrator` skill 生成文中插图
- **爆文分析**：使用 `viral-article-analyzer` skill 分析同类爆文的写作模式，为选题和写法提供参考
