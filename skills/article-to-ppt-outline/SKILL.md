# Article to PPT Outline Generator

将输入文章转化为结构化 PPT 大纲，每页含【视觉排版建议】【文案内容】【讲师逐字稿】三个模块。

## 输入规范

- `source_text`（必填）：原始参考资料（纯文本、Markdown 等格式均可）
- `target_pages`（选填）：期望总页数，默认根据内容体量自动评估（通常 10–30 页）
- `audience`（选填）：目标听众特征（如技术人员、高管、销售），默认通用职场人群

## 工作流程

严格按顺序执行以下 4 步：

**步骤 1：知识原子化抽取**
完整阅读 source_text，滤除冗余过渡句和修饰语，将长文本拆解为相互独立、完全穷尽的原子知识点（Bullet points）。

**步骤 2：规划宏观骨架**
根据知识点的逻辑关系，构建 PPT 整体架构（如：痛点引入 → 核心概念 → 落地实操 → 总结行动）。确保逻辑递进，控制单页信息密度。

**步骤 3：逐页精细扩写**
按照骨架逐页生成内容，每页必须包含三个固定模块（见输出格式）。

**步骤 4：合规自检**
检查屏幕文案是否超字数、逐字稿是否口语化、是否存在原文中不存在的幻觉信息。

## 输出格式

每页严格使用以下 Markdown 模板：

```
### 第 X 页：[单页主标题]

**【视觉与排版建议】**
* **排版结构**：[如：左右对比、四宫格、雷达图、极简居中等]
* **配图/图标建议**：[具体描述需要的视觉元素]

**【文案内容】**
* [小标题 1]：[提炼后的极简短句，不超过 15 个字]
* [小标题 2]：[提炼后的极简短句]
（注：本部分内容必须高度精简，能用词的不用句，能用表的不用段落）

**【讲师逐字稿】**

> "[口语化表达。将书面语转化为对话口吻，有起承转合，适当加入'大家看屏幕'、'举个例子'等连接词。字数控制在演讲 1–2 分钟的体量。]"
```

## 约束规则

- **严禁照搬原文**：禁止将大段原文直接复制到【文案内容】，屏幕文案必须经过压缩和结构化
- **严禁缺漏逐字稿**：每一页都必须有【讲师逐字稿】
- **严禁信息幻觉**：只能基于 source_text 包含的信息提炼，不可编造原文未提及的数据或专业术语
- **语言风格**：输出默认使用简体中文；逐字稿需体现专业、自信且互动的讲师状态
